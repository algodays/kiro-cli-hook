kiro-plan-mode

A plan-first discipline layer for Kiro agents — built entirely with Kiro Hooks

Kiro Birthday Week 2026 Day 1: Build a Hook

The problem I kept hitting

Kiro's hooks are a great automation primitive.

But the agent would jump straight to editing files the moment I asked for a feature —

sometimes before it had fully understood the codebase.

The diff was usually fine. The plan behind it was implicit.

I wanted the agent to think before it wrote code — and I wanted to approve that plan first.

So I built it with hooks

1Agent asked to build something → blocked by hook
2Agent must explore the codebase & write a plan first
3User reviews the plan, replies "approve plan"
4Agent executes only after approval

No fork. No plugin. Just four JSON files in .kiro/hooks/.

Four hooks

plan-first-gate

UserPromptSubmit · shell

Blocks mutating prompts until a plan is written and approved. Read-only questions pass through.

skill-auto-loader

SessionStart · shell

Detects project type from marker files and injects the matching skill markdown into context.

todo-tracker

Stop · agent prompt

Updates .kiro/plan/TODO.md after every agent turn with completed and pending tasks.

secret-guard

PreToolUse · agent prompt

Scans for API keys, tokens, private keys, and dangerous commands before any write.

Live: a mutating prompt gets blocked

$ kiro-cli chat "add a dark mode toggle to the demo app"
> reading demo-app/public/index.html...
> reading demo-app/src/server.js...
> reading demo-app/package.json...
PLAN MODE ACTIVE — request blocked
the agent must not edit files yet.
writing plan to .kiro/plan/PLAN.md...
plan written — status: awaiting approval
reply "approve plan" to unlock execution.

Live: approve, then execute

$ kiro-cli chat "approve plan"
plan approved — execution mode unlocked.
$ kiro-cli chat "implement the dark mode toggle now"
> reading approved plan...
> modifying demo-app/public/index.html...
> +92 lines: CSS variables, dark theme, toggle button, localStorage
dark mode toggle implemented.
source files modified only after explicit user approval.

plan first. then build.
